## 概要

**Manael v3.0** のリリースをお知らせします。このメジャーリリースでは、画像処理、依存関係の管理、クエリパラメータのサポートに関して大きな変更が加えられています。

## libvips への移行

Manael v3.0 では、画像処理の基盤ライブラリを [`bimg`](https://github.com/h2non/bimg) バインディングを通じて [libvips](https://www.libvips.org/) へ移行しました。libvips は高速かつメモリ効率の高い画像処理ライブラリであり、以前の実装と比べてパフォーマンスの向上と対応フォーマットの拡大を実現しています。

### 動的リンクの要件

この移行により、**実行時にホストシステムへ libvips のインストールが必要**になりました。Manael は完全な静的リンクバイナリではなくなっています。

**Debian/Ubuntu の場合:**

```sh
sudo apt-get install -y libvips-dev
```

**Alpine Linux の場合:**

```sh
apk add vips-dev
```

公式 Docker イメージには libvips が含まれているため、追加のセットアップは不要です。

## 新しいクエリパラメータ

Manael v3.0 では、オンザフライの画像リサイズに対応した新しいクエリパラメータが追加されました。

| パラメータ | 説明                                      | 例             |
|-----------|-------------------------------------------|----------------|
| `w`       | ターゲット幅（ピクセル）                  | `?w=800`       |
| `h`       | ターゲット高さ（ピクセル）                | `?h=600`       |
| `fit`     | リサイズのフィットモード（`cover`、`contain` など） | `?fit=cover` |

これらのパラメータを組み合わせることで、画像のスケーリングおよびクロップ方法を制御できます。

## 破壊的変更

- バイナリの実行に、ホスト上で `libvips` が必要になりました。
- Go の最小バージョンが更新されました。現在の要件については `go.mod` ファイルをご確認ください。

## アップグレード方法

1. ホストに `libvips` をインストールするか、コンテナイメージを更新してください。
2. Manael バイナリを v3.0 リリースのものに置き換えてください。
3. URL ベースの画像変換を利用している場合は、クエリパラメータのドキュメントをご確認ください。

変更の全一覧については [CHANGELOG](https://github.com/manaelproxy/manael/blob/main/CHANGELOG.md) をご参照ください。
